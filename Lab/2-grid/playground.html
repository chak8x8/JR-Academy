Why use <main>/<section> if <div> works?
  • Screen readers can jump straight to main → better accessibility.
  • Search engines understand structure → better semantics/SEO.
  • Doesn’t change style; just adds meaning.

3) Grid with gutters + N columns (toggle everything)

What do these toggles change?
  • Columns (12/14) — changes how many content columns exist. 12 divides nicely (2,3,4,6).
  • Card span — how many columns each card takes. e.g., span 4 in a 12-col grid → 3 per row.
  • Gutters — the side 1fr columns. They share leftover space (two gutters → each gets half).
  • Use minmax(auto,60px) — auto minimum fits content width (may resist shrinking); 0 min is more flexible.
  • Show outlines — draws dashed boxes so you can see the frame/content regions.

3.5) :hover (pseudo-class), ::before/::after (pseudo-elements), cursor, transition & transform

On :hover, we move the button up (transform: translateY(-4px)) and change its color. The transition makes it smooth.
What is :hover? What is &:hover in SCSS?

:hover is a pseudo-class — a temporary state (“mouse is over this”).

In SCSS you often see:

.button {
  // SCSS nesting
  &:hover { transform: translateY(-4px); }
}

It compiles to plain CSS:

.button:hover { transform: translateY(-4px); }

Tip: pair :hover effects with transition for smooth animations.

What does cursor: pointer do?

It changes the mouse cursor to a hand shape to tell users “this is clickable”. It does not make things clickable by itself — you still need a link (<a>) or JS on a button/div.

What do transition and transform do?
  • transform moves/scales/rotates the element (e.g., translateY(-4px)).
  • transition animates the change smoothly: transition: transform 0.2s ease;
  • Good pattern: animate specific properties (transform, opacity) instead of all for better performance.

4) Grid lines — “Start at column 2, span 12”

Think of vertical lines numbered left→right. You can say: start at line 2, take 12 columns.
grid-column: 1 / -1 (full width)
grid-column: 2 / span 12 (start at line 2, take 12)
grid-column: 1 / span 1 (just the first column)

Summary (plain English)

  • <main> = the main story. <section> = a chapter. <div> = plain box.
  • Semantic tags don’t change visuals, but help screen readers & SEO.
  • display: block (stack), inline-block (side-by-side), flex (1D rows/cols), grid (2D rows+cols).
  • grid-template-columns defines how many columns and their sizes.
  • 1fr gutters share leftover space; two gutters → each gets half.
  • Why 12? Easy divisors (2,3,4,6) → flexible rows.
  • grid-column: 2 / span 12 means “start at line 2 and occupy 12 columns”.
  • :hover is a state; ::before/::after are decorative boxes; cursor changes mouse pointer; transition + transform animate motion smoothly.
