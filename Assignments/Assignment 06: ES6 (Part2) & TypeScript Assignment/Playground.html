Concept cheat‑sheet

Callbacks

A callback is a function you pass into another function to be called later. We simulate network delay with setTimeout.

// Shape
function doWork(arg, callback) {
  setTimeout(() => callback("done"), 2000);
}
  • Pros: simple shape, built into many browser APIs.
  • Cons: nesting can get messy ("callback hell").
Promises

A Promise represents a value you will have later. Create with new Promise((resolve, reject) => { ... }) and consume with .then/.catch.

// Shape
function work() {
  return new Promise((resolve, reject) => {
    setTimeout(() => resolve("ok"), 2000);
  });
}
work().then(v => console.log(v)).catch(e => console.error(e));
Async/Await

async functions return Promises. await pauses inside an async function until the Promise settles. Use try/catch for errors.

async function run(){
  try {
    const v = await work();
    console.log(v);
  } catch (e) {
    console.error(e);
  }
}
Bonus: TypeScript (read‑only note)

TypeScript adds types. You write .ts, compile with tsc to .js. Quick snippet:

// index.ts
function add(a: number, b: number): number { return a + b; }
interface Person { name: string; age: number; }
function greet(p: Person): void { console.log(`Hello, ${p.name}!`); }

This page does not compile TS in the browser; use your terminal with npm i -g typescript and tsc --init.
